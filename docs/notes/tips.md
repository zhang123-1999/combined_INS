### 📝 Antigravity / ESKF 系统性调试提示词（MD版）

**提示词目标：** 请协助我排查在 ESKF (Error-State Kalman Filter) 导航框架下，纯惯导或仅带 ODO/NHC 约束时出现的“高频速度与姿态震荡”问题。

---

#### 1. 惯导解算（Mechanization）逻辑检查

* **重力补偿方向：** 检查速度更新方程：`V_dot_n = C_b_n * f_b + g_n`。在 ENU 坐标系下，`g_n` 必须是 `[0, 0, -9.8]`。请确认是“加上”重力向量还是“减去”。如果符号写反，系统会像倒立摆一样产生发散性震荡。
* **四元数更新与归一化：** 在 50Hz 采样率下，姿态积分是否使用了旋转矢量（Rotation Vector）？确认四元数在每一帧更新后都执行了 `normalize()`。未归一化的四元数会导致 `C_b_n` 矩阵失去正交性，从而在重力投影时产生虚假加速度。
* **哥氏力项（Coriolis）：** 检查公式中是否包含了 `-(2 * w_ie_n + w_en_n) x V_n`。请尝试在代码中将此项置零，观察震荡是否消失。若该项的正负号或叉乘顺序错误，会导致速度与姿态之间产生不稳定的正反馈。

#### 2. 误差状态传播（F 矩阵）一致性

* **雅可比矩阵 F 的符号：** 检查速度误差对姿态误差的偏导项 `dV_dot / d_phi`。
* 标准公式应为：`-[f_n x]`（即导航系下加速度的反对称阵的前面带负号）。
* 如果这个负号写错或漏写，滤波器在修正姿态时会向反方向补偿，导致越补越偏。


* **噪声驱动 Q 阵：** 检查 IMU 的噪声参数（ARW, VRW）。如果  阵设置得过小，滤波器会变得过于“僵硬”，在反馈修正时会产生类似数值过冲的锯齿状震荡。

#### 3. 约束观测模型（ODO/NHC）

* **杠杆臂（Lever Arm）：** 检查公式 `V_imu = V_wheel + w_b x r_b`。
* 确认杠杆臂向量 `r_b` 的方向（是从 IMU 指向车轮中心，还是反向）。
* 确认角速度 `w_b` 参与叉乘的符号。


* **安装偏角（Boresight）：** 检查 IMU 到车体系的旋转矩阵。如果 NHC 认为侧向速度应为 0，但由于安装角误差导致前进速度分量被当成了侧向误差，系统会强行扭转航向角来消除这个“误差”，引发震荡。
* **观测噪声 R 阵：** 请尝试将 NHC（侧向/垂向速度）的观测标准差（Sigma）临时调大 10 倍。如果震荡频率随之改变或消失，说明是 Kalman 增益过大导致的控制系统失稳。

#### 4. 状态复位与反馈（The Reset Step）

* **状态注入（Injection）：** 检查 `state = state + delta_x` 这一步。
* 姿态修正：`q = q * exp(delta_theta)`。请确认四元数乘法的顺序（左乘还是右乘）与你的 ESKF 推导一致。
* 如果修正量 `delta_theta` 的方向与名义状态更新方向相反，每一帧反馈都会给系统一个相反的冲击。


* **误差清零：** 确认在每次 `Update` 之后，误差状态向量 `delta_x` 是否被立即执行了 `setZero()`？如果没有清零，旧的误差会在下一帧被重复补偿。
